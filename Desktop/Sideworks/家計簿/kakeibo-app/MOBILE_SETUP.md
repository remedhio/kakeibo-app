# モバイルでアプリを実行する方法

## 方法1: Expo Goアプリを使う（推奨・開発中に最適）

Expo Goは、Expoアプリを開発中にモバイルデバイスで簡単にテストできる公式アプリです。

### ステップ1: Expo Goアプリをインストール

#### iOSの場合
1. App Storeで「Expo Go」を検索
2. インストール

#### Androidの場合
1. Google Play Storeで「Expo Go」を検索
2. インストール

### ステップ2: 開発サーバーを起動

プロジェクトディレクトリで以下を実行：

```bash
cd kakeibo-app
npm start
```

または

```bash
npx expo start
```

### ステップ3: モバイルデバイスで接続

#### オプションA: QRコードで接続（最も簡単）

1. ターミナルにQRコードが表示されます
2. **iOS**: カメラアプリでQRコードをスキャン → Expo Goが開きます
3. **Android**: Expo Goアプリを開き、「Scan QR code」をタップ → QRコードをスキャン

#### オプションB: 同じWi-Fiネットワークで接続

1. PCとモバイルデバイスが同じWi-Fiネットワークに接続されていることを確認
2. ターミナルで `i`（iOS）または `a`（Android）を押す
3. 自動的にExpo Goアプリが開きます

#### オプションC: 手動でURLを入力

1. ターミナルに表示されているURL（例: `exp://192.168.1.100:8081`）をコピー
2. Expo Goアプリを開き、「Enter URL manually」をタップ
3. URLを貼り付けて「Connect」をタップ

### トラブルシューティング

#### 接続できない場合

1. **同じWi-Fiネットワークに接続されているか確認**
   - PCとモバイルデバイスが同じWi-Fiに接続されている必要があります

2. **ファイアウォールを確認**
   - PCのファイアウォールがポート8081をブロックしていないか確認

3. **トンネルモードを使う**
   ```bash
   npx expo start --tunnel
   ```
   - これにより、異なるネットワークでも接続可能になります（少し遅くなる場合があります）

4. **LANモードを使う**
   ```bash
   npx expo start --lan
   ```
   - ローカルネットワーク経由で接続します

## 方法2: 開発ビルドを作成する（本番環境に近い）

より本番環境に近い形でテストしたい場合、開発ビルドを作成できます。

### iOS（Macのみ）

```bash
# iOSシミュレーターで実行
npm run ios

# 実機で実行（Xcodeが必要）
npx expo run:ios --device
```

### Android

```bash
# Androidエミュレーターで実行（Android Studioが必要）
npm run android

# 実機で実行
npx expo run:android --device
```

### 前提条件

- **iOS**:
  - Macが必要
  - Xcodeが必要（App Storeからインストール）
  - Apple Developerアカウント（実機で実行する場合）

- **Android**:
  - Android Studioが必要
  - Android SDKが必要
  - USBデバッグが有効なAndroidデバイス（実機で実行する場合）

## 推奨される開発フロー

1. **開発中**: Expo Goアプリを使用（最も簡単・高速）
2. **テスト**: 開発ビルドを作成して実機でテスト
3. **リリース**: EAS Buildで本番ビルドを作成

## 注意事項

### Expo Goの制限

- 一部のネイティブモジュールが使用できない場合があります
- カスタムネイティブコードは使用できません
- 本アプリは標準的なExpo機能のみを使用しているため、Expo Goで問題なく動作します

### パフォーマンス

- Expo Goは開発用アプリのため、本番ビルドより若干遅い場合があります
- 本番環境では必ず開発ビルドまたは本番ビルドを使用してください
